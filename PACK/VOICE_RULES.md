# Voice Rules — Petty

## Purpose
Define tone and language rules for all product copy.

## Tone
- Direct, Warm, Confident

## Triggers
- All user-facing UI copy
- Onboarding flows
- Error messages
- Email notifications
- Marketing site

## Dos
- Use active voice
- Short, clear sentences
- Talk to the user directly ("you", "your")
- Use plain language — no jargon
- Lead with the action or outcome

## Don'ts
- No passive voice ("has been sent" → "we sent")
- No corporate filler ("leverage", "seamless", "synergy")
- Don't over-apologise in error states
- No ellipsis in UI copy

## Example Copy

### Empty State
- Good: "No pet yet. Create your first one."
- Bad: "It seems you haven't created anything at this time."

### Error Message
- Good: "Something went wrong. Try again."
- Bad: "An unexpected error has occurred."

### CTA
- Good: "Get started"
- Bad: "Click here to begin your free trial experience"
